Operational Overview

This dashboard is designed to help your team move from raw trace collection to confident operational decisions. Use the KPI row for fast health checks, the charts for trend analysis, and the request panels below to investigate expensive prompts, latency outliers, and response-quality drift.

Observe
What is happening right now?

Use requests, latency, error rate, and evaluation score to understand current system health.

Diagnose
What is causing issues?

Inspect slow requests, costly prompts, and model-specific behavior to isolate the source.

Decide
What should we change next?

Compare speed, spend, and evaluation score together before tuning prompts or switching models.

How To Read This
KPI cards

Your fastest summary of traffic, cost, latency, evaluation score, and failure rate.

Charts

Useful for spotting trend shifts, concentration by model, and traffic shape over time.

Request panels

Best for drilling into concrete examples when you need to explain a spike or regression.

Model Usage

Shows where request volume is concentrated so you can identify dominant models and traffic skew.

Latency Distribution

Breaks traffic into fast, medium, and slow buckets to surface user-experience risk quickly.

Requests Over Time

Use this trend line to understand load changes, rollout impact, and traffic surges over time.

Tokens & Cost

Compares model token usage and estimated spend so you can spot inefficient patterns before they grow.

Costly Prompts

Highlights prompts with the highest per-request cost so you can review prompt size and model choice.

Slow Requests

Surfaces the worst latency outliers so your team can investigate degraded user experience immediately.

What This Dashboard Helps You Answer
Why did response quality change?

Compare evaluation badges, recent traces, and expensive or slow requests to see what shifted.

Which model is driving spend?

Use model usage, token trends, and costly prompts together to spot cost concentration.

Are users waiting too long?

Check P95, latency distribution, and the slow requests panel for tail latency problems.

Is the rollout healthy?

Watch requests over time, error rate, and evaluation score to catch regressions early.

Operator Notes
Evaluation score

A heuristic signal that helps flag weak or strong responses. Use it directionally, not as a final judgment.

Cost data

Cost is estimated from captured request metadata, so it is best used for trend comparison and outlier detection.

Live updates

The dashboard refreshes automatically, making it suitable for active monitoring during releases or incidents.

Recent Requests

The latest captured traces with latency, quality, status, token usage, and cost for quick request-level inspection.
